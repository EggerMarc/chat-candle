# chat-candle web example (Vite + React)

In-browser chat with Qwen3-0.6B (Q4_K_M GGUF) running on `chat-candle` compiled
to wasm. Weights **auto-download** from the HF CDN on first load and cache in the
browser (Cache API), so subsequent loads are instant. Inference runs in a **Web
Worker** so the UI stays responsive and tokens stream live.

> candle 0.10.2 has no wasm WebGPU backend, so this is the **CPU** path — correct
> but slow. It proves the browser pipeline; speed waits on WebGPU.

## Run

```bash
# 1. build the wasm bindings into src/pkg (run from this dir)
npm run build:wasm
# 2. install + dev server
npm install
npm run dev        # open the printed localhost URL
```

`npm run build:wasm` is `wasm-pack build ../.. --target web --out-dir
examples/web/src/pkg --dev -- --no-default-features`. Drop `--dev` for an
optimized (smaller, faster) wasm once you have `wasm-opt`.

## How it fits together

- `src/worker.ts` — fetches + caches weights, `init()`s the wasm, builds
  `LocalChat`, and runs `generate()` (a synchronous wasm call) off the main
  thread, posting tokens back.
- `src/App.tsx` — spawns the worker on mount, shows download progress, renders
  the streaming chat.
- `src/pkg/` — generated by `wasm-pack` (gitignored).

To use a different model, edit `GGUF_URL` / `TOKENIZER_URL` in `src/worker.ts` —
any Llama/Qwen/Mistral-family GGUF works (the loader reads the architecture from
GGUF metadata).
